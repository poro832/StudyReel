### 1:45 — 5. 어떻게 구현했나 (아키텍처) *(30초)*
> 구조는 **3개 레이어 단방향 의존**입니다. UI는 Riverpod Provider만 바라보고,
> Provider는 Repository만 압니다. 모든 데이터 정책 — **"Firestore 캐시를 먼저 보고, 없을 때만
> YouTube API를 호출한다"** — 는 Repository 한 곳에만 존재합니다.
> 디렉토리도 core, data, domain, presentation으로 레이어 그대로 나눴고,
> 이 규칙은 AGENTS.md에 불변식으로 박아 AI 에이전트도 어기지 못하게 했습니다.

`→ 다음`

### 2:15 — 6. 구현 시행착오 (error 152) *(30초)*
> 최대 난관은 **error 152**, 쇼츠 인앱 재생 차단이었습니다. API는 "임베드 가능"이라고 답해놓고 런타임에 막습니다.
> 처음엔 "WebView 합성 불가", "autoplay 차단"으로 오진했지만 — 실기기에서 **에러 UI가 또렷이 렌더되는 걸 보고**
> 렌더링 문제가 아니라 콘텐츠 정책임을 규명했습니다.
> 라이브러리 소스를 열어 `origin`이 `host`로도 쓰인다는 걸 찾았고, 호스트를 **youtube-nocookie로 교체해 돌파**,
> 실기기에서 재생을 검증했습니다.

`→ 다음`

### 2:45 — 7. 성능 최적화 · 코드 품질 *(20초)*
> 성능은 **쿼터 보호**가 핵심입니다. Firestore 영구 캐싱으로 재방문 시 API 호출 0회,
> 429는 지수 백오프, 재생 불가 영상은 3초 워치독으로 자동 스킵합니다.
> 품질은 **analyze 경고 0건**을 게이트로 유지하고, 공통 위젯 재사용과 데드코드 즉시 제거를 원칙으로 했습니다.

### 2. 질의응답 (5점) — 답변 훈련 스크립트

**Q. ADR 보여주세요 (최소 3개)**
> 슬라이드 10을 띄우고: "5건 있습니다. 핵심 3개만 말씀드리면 — ADR-0001 Flutter는 단일 코드베이스로 세로 피드를 빠르게 만들기 위해, ADR-0003 Cloud Functions 미사용은 Blaze 과금을 피하고 클라이언트 직접 호출 + 캐싱으로 대체하기 위해, ADR-0004 피벗은 Gemini 토큰 한도를 실측한 데이터를 근거로 결정했습니다."

**Q. 앱의 구조는? (레이어/디렉토리)**
> "3레이어 단방향입니다. presentation은 domain의 Riverpod Provider만 watch하고, domain은 data의 Repository만 호출합니다. 디렉토리도 core / data(models·repositories·services) / domain / presentation으로 레이어 그대로입니다. UI에서 Firestore나 HTTP를 직접 호출하는 일은 AGENTS.md 불변식으로 금지했습니다." *(백업 슬라이드 B1)*

**Q. 개발 환경 설정은?**
> "docs/setup.md 기준 5분 목표입니다. Flutter SDK 확인 → pub get → Google Cloud Console에서 YouTube API 키 발급 → flutterfire configure로 Firebase 연결 → dart-define으로 키 주입해 flutter run. 한글 경로에서 impellerc가 크래시하는 이슈는 buildDirectory를 ASCII 경로로 리다이렉트해 우회했고 이것도 문서화했습니다."

**Q. 빌드와 배포는 어떤 단계로?**
> "빌드는 소스를 APK라는 실행물로 컴파일하는 단계 — flutter build apk --release에 API 키를 dart-define으로 주입합니다. 배포는 그 실행물을 사용자에게 전달하는 단계 — 저희는 서버가 없으므로(ADR-0003) APK를 GitHub Pages에 올려 다운로드 + adb install로 설치합니다. 빌드 전 analyze 0건과 테스트 전체 통과를 체크리스트로 강제합니다."

**Q. (그 외 모르는 질문)**
> "그 부분은 정확히 모릅니다. 추측으로는 ~같습니다만, 확인 후 알려드리겠습니다." — 모른다고 인정 + ADR/문서 위치 언급이 추측 답변보다 낫다.
