## 1. 앱 개요

**한 줄 정의**  
관심사/전공을 설정하면, AI가 생성한 짧은 학습 카드를 릴스처럼 스와이프하며 소비하고 — 더 깊이 보고 싶으면 관련 유튜브 쇼츠로 연결되는 모바일 학습 앱

**문제 정의**  
- 기존 학습 앱은 강제·차단 방식(Forest, 스크린타임)이거나 딱딱한 커리큘럼 형태
- 릴스·쇼츠처럼 짧고 중독성 있는 포맷으로 학습 콘텐츠를 소비할 수 있는 앱이 없음
- 유튜브에 교육 쇼츠는 있지만, 관심사 기반으로 큐레이션·정리해주는 앱이 부재

**타겟 사용자**  
SNS·유튜브에 시간을 쓰면서 "공부할걸"이라는 후회를 느끼는 대학생/청년층

## 2. 핵심 화면 & 사용자 흐름

### 화면 목록

| 화면 | 역할 |
|---|---|
| 온보딩 | 관심사/전공 3~5개 선택 |
| 메인 피드 | 세로 스와이프 카드 (릴스 스타일) + 북마크 버튼 |
| 카드 상세 | 핵심 개념 + 3줄 요약 → **퀴즈 1문제** → 유튜브 쇼츠 버튼 |
| 검색/탐색 | 토픽별 카드 브라우징 |
| 프로필 | 오늘 본 카드 수, **스트릭**, 관심사 수정 |

### 사용자 흐름

```
앱 시작
  → 온보딩: 관심사 3~5개 선택
  → 메인 피드 (세로 스와이프)
      → 위로 스와이프: 다음 카드
      → 아래로 스와이프: 이전 카드
      → 북마크 버튼: 카드 저장 (Firestore bookmarks 필드)
      → 카드 탭: 상세 보기
          → 핵심 개념 + 요약
          → [NEW] 퀴즈 1문제 (객관식 4지선다, Claude가 생성)
              → 정답 시: "정답! 🎉" + 유튜브 버튼 활성화
              → 오답 시: 힌트 표시 + 재시도 or 스킵
          → 유튜브 쇼츠 3개 목록
          → url_launcher로 YouTube 앱 실행
```

### 리텐션 구조

| 기능 | 효과 | 구현 난이도 |
|---|---|---|
| **퀴즈** | 유튜브 이탈 전 체류시간 증가 + 학습 확인 | 낮음 (Claude 프롬프트 추가) |
| **스트릭** | 연속 학습 동기 부여 | 낮음 (Firestore 날짜 기록) |
| **북마크** | 재방문 이유 생성 | 매우 낮음 (필드 1개 추가) |

## 3. 기술 스택 & 아키텍처

### 전체 구조

```
[Flutter 앱]
    │
    ├── Firebase Auth        ← Google 소셜 로그인
    ├── Firestore            ← 카드 캐시, 사용자 관심사 저장
    └── Cloud Functions      ← API 키 보호용 서버리스 미들웨어
            │
            ├── Claude API           ← 학습 카드 텍스트 생성
            └── YouTube Data API v3  ← 관련 쇼츠 검색
```

### Flutter 앱 레이어 구조

```
lib/
├── presentation/   ← 화면 UI (피드, 카드, 프로필)
├── domain/         ← 비즈니스 로직 (카드 생성 요청, 관심사 관리)
├── data/           ← API 통신, Firestore 읽기/쓰기
└── core/           ← 공통 유틸, 라우터, 테마
```

### 카드 생성 흐름

```
관심사 설정
  → Cloud Function 호출
  → Claude API: "컴퓨터공학 관련 학습 카드 5개 JSON으로 생성"
  → 결과를 Firestore에 캐시
  → 앱 피드에 표시
```

### YouTube 연동 흐름

```
카드 상세 탭
  → 카드 키워드로 YouTube Data API 검색
  → 교육용 쇼츠 상위 3개 링크 반환
  → url_launcher 패키지로 YouTube 앱 실행 (WebView 미사용)
```

## 4. 기술 선택 근거

### Flutter
- iOS/Android 동시 지원 → 발표 데모에서 두 플랫폼 모두 시연 가능
- 세로 스와이프 피드(PageView)가 Flutter에서 구현이 간단함
- Dart는 학습 비용이 낮고, 6주 일정에 적합
- 대안인 React Native보다 UI 성능이 안정적

### Firebase
- **Auth**: Google 로그인을 5줄로 구현 가능, 별도 백엔드 없이 인증 처리
- **Firestore**: 스키마 없이 빠르게 카드 데이터 저장/조회 가능
- **Cloud Functions**: Claude/YouTube API 키를 앱에 노출하지 않고 서버리스로 보호
- 직접 백엔드(Node/Django) 대비 설정 시간 대폭 절약

### Claude API
- 자연스러운 한국어 학습 카드 생성에 최적화
- JSON 출력 포맷 지정이 용이 → 앱에서 파싱 편함
- 긴 문맥 처리가 안정적

### YouTube Data API v3
- 이미 전 세계 교육 쇼츠가 대규모로 존재 → 콘텐츠 수집 비용 없음
- 무료 쿼터(일 10,000유닛) 내에서 MVP 운영 가능
- 직접 영상 업로드 방식 대비 스토리지/인프라 비용 없음

## 5. 에러 처리

| 상황 | 대응 |
|---|---|
| Claude API 응답 실패 | 캐시된 이전 카드 표시 + "새로고침" 버튼 |
| YouTube API 쿼터 초과 | 유튜브 버튼 비활성화 + "검색 제한 도달" 안내 |
| 네트워크 없음 | Firestore 오프라인 캐시로 기존 카드 계속 표시 |
| 로그인 실패 | 재시도 버튼 + 게스트 모드 허용 |

## 6. 테스트 전략

| 레벨 | 대상 | 도구 |
|---|---|---|
| 단위 테스트 | 카드 파싱 로직, 관심사 필터링 | Flutter `flutter_test` |
| 통합 테스트 | Claude API → 카드 생성 → Firestore 저장 흐름 | Firebase Emulator |
| UI 테스트 | 피드 스와이프, 카드 탭 → 유튜브 연결 | Flutter Integration Test |

**실행 명령**
```bash
# 단위 + 통합 테스트
flutter test

# UI 통합 테스트
flutter test integration_test/
```
