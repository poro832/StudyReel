# BONUS.md — 가산점 항목 정리

기본 요구사항(관심사 선택 → 학습 피드 → 저장) 외에 추가로 구현한 항목.

## A. 실시간 외부 API 연동 + 쿼터 회복 설계

- YouTube Data API v3를 클라이언트에서 직접 연동 (Cloud Functions 무과금 우회).
- **쿼터 장애를 설계로 흡수**:
  - YouTube 403(일일 한도) → Firestore 영구 캐싱으로 재호출 0회화
  - Gemini 429 → 지수 백오프(10→20→40초) 자동 재시도
- 근거 문서: `ADR-0003`, `ADR-0004`.
- 가치: 무료 한도 환경에서도 데모가 끊기지 않는 견고성.

## B. 제품 피벗을 데이터로 판단하고 데드코드까지 정리

- Phase 1(Gemini 생성 카드) → Phase 2(YouTube Shorts 피드)로 전환.
  실사용 토큰 한도 문제를 근거로 한 의사결정.
- 전환 후 **고아 코드 7개 파일 + 미사용 의존성(`cloud_functions`) 제거**,
  `flutter test` 그린(15건) 유지.
- 근거 문서: `ADR-0004`, `ADR-0005`.
- 가치: 단순 기능 추가가 아닌, 추적 가능한 아키텍처 진화 + 부채 청산.

## C. 학습 지속성 기능 (스트릭) + 결정적 테스트

- `StreakRepository`: 첫 방문 / 같은 날 / 연속일 +1 / 공백 시 리셋 로직.
- 시간 의존 로직을 `now` 콜백 주입으로 **결정적 단위 테스트** (5케이스).
- 프로필 화면에서 🔥 연속 학습일 + 저장한 영상 목록 시각화.
- 가치: 단순 CRUD를 넘어선 사용자 리텐션 기능 + 테스트 가능한 설계.
